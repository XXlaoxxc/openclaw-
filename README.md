## 🚀 使用方法

### 1. 克隆仓库
```bash
git clone https://github.com/XXlaoxxc/openclaw-.git
cd openclaw-
```

### 2. 配置敏感信息
编辑 `config/openclaw.json`，将占位符替换为实际值：
- `YOUR_GITHUB_TOKEN_HERE` → GitHub Personal Access Token
- `YOUR_QWEN_API_KEY_HERE` → 通义千问 API 密钥  
- `YOUR_FEISHU_APP_ID_HERE` / `YOUR_FEISHU_APP_SECRET_HERE` → 飞书应用凭证
- `YOUR_GATEWAY_AUTH_TOKEN_HERE` → Gateway 认证令牌

### 3. 部署配置
```bash
cp config/openclaw.json ~/.openclaw/
chmod 600 ~/.openclaw/openclaw.json
```

### 4. 部署工作区模板
根据需要复制工作区模板：
```bash
cp -r workspace-templates/workspace-xiaoyan-expert ~/.openclaw/
```

### 5. 安装插件
根据 `plugins.md` 重新安装插件：
```bash
openclaw plugins install feishu-openclaw-plugin
openclaw plugins install dingtalk-connector  
openclaw plugins install qqbot
openclaw plugins install wecom-openclaw-plugin
```

### 6. 创建完整备份
使用提供的脚本创建包含所有数据的本地备份：
```bash
chmod +x scripts/create_backup.sh
./scripts/create_backup.sh
```

## 🔒 安全最佳实践

- **密钥管理**: 使用环境变量或密钥管理服务存储敏感信息
- **权限控制**: 配置文件权限应设为 600 (`chmod 600`)
- **定期轮换**: 定期更新 API 密钥并重新部署
- **版本控制**: 只提交配置结构，不提交实际密钥
